Install

Use the release binary installer unless you are developing inside this repository.

# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/SamuelDBines/nunchucks/main/scripts/install-cli.sh | sh

# install a specific release
curl -fsSL https://raw.githubusercontent.com/SamuelDBines/nunchucks/main/scripts/install-cli.sh | sh -s v0.0.13

# verify
nunchucks version
nunchucks help
# Windows PowerShell
irm https://raw.githubusercontent.com/SamuelDBines/nunchucks/main/scripts/install-cli.ps1 | iex

# verify
nunchucks.exe version
nunchucks.exe help
# local source build from this repository
cd go
go build -o ~/.local/bin/nunchucks ./cmd/nunchucks

Release binaries are published on GitHub Releases.

CLI Overview

The CLI renders one template to stdout or precompiles a views tree to static output. Watch mode rerenders on template file changes and keeps running after render errors so you can fix files without restarting.

nunchucks renders Nunchucks templates from the command line.

Usage:
  nunchucks <command> [options]

Commands:
  render      Render one template to stdout
  precompile  Render a views directory to static output
  version     Print CLI version information
  help        Show general help or help for a command

Examples:
  nunchucks render -views ./views -template index.njk -data '{"title":"Hello"}'
  nunchucks precompile -views ./views -out ./public
  nunchucks precompile -views ./views -out ./public --watch
  nunchucks help render
  nunchucks version

Render Command

Render one template file from the views directory and print the result to stdout.

Usage:
  nunchucks render [options]

Render a single template and write the result to stdout.

Options:
  -views string        templates directory (default "views")
  -template string     template path relative to views (required)
  -data string         JSON context object (default "{}")
  -global value        global template, repeatable
  -global-head value   global head template, repeatable
  -global-foot value   global foot template, repeatable

Example:
  nunchucks render -views ./views -template index.njk -data '{"user":{"name":"sam"}}'
nunchucks render   -views ./views   -template invoice.njk   -data '{"invoice":{"number":"A-102","total":99.5}}'

Precompile Command

Render an entire views tree to disk. By default it preserves template filenames. Use -out-format html when you want rendered .html files from .njk templates.

Usage:
  nunchucks precompile [options]

Render a views directory to static files on disk.

Options:
  -views string        templates directory (default "views")
  -out string          output directory (default "public")
  -out-format string   output naming: preserve or html (default "preserve")
  -watch               rerender when template files change
  -interval duration   polling interval for watch mode (default 1s)
  -data string         JSON context object (default "{}")
  -global value        global template, repeatable
  -global-head value   global head template, repeatable
  -global-foot value   global foot template, repeatable

Example:
  nunchucks precompile -views ./views -out ./public -data '{"title":"Hello"}'
  nunchucks precompile -views ./views -out ./public -out-format html
  nunchucks precompile -views ./views -out ./public --watch -interval 750ms
# preserve source filenames
nunchucks precompile -views ./views -out ./public

# convert .njk output targets to .html
nunchucks precompile -views ./views -out ./public -out-format html

# watch and rerender on change
nunchucks precompile -views ./views -out ./public --watch

# slower polling interval for larger trees
nunchucks precompile -views ./views -out ./public --watch -interval 2s

Examples

# inspect installed version
nunchucks version

# render one file to stdout
nunchucks render -views ./views -template page.njk -data '{"title":"Docs"}'

# write a static site build to ./public
nunchucks precompile -views ./views -out ./public -out-format html

# keep rebuilding while editing partials and layouts
nunchucks precompile -views ./views -out ./public -out-format html --watch
